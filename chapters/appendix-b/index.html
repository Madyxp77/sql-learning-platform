Appendix B: Glossary of SQL and Data Terms

This glossary defines key terms and concepts related to SQL and data analysis. It serves as a handy reference to clarify terminology encountered throughout "The SQL Path."

Aggregate Function
A function that performs a calculation on a set of values (a column) and returns a single value. Common examples include COUNT(), SUM(), AVG(), MIN(), and MAX().
Alias
A temporary, alternative name given to a table or a column in a SQL query. Used to make column names more readable or to shorten table names for easier referencing, especially in JOINs.
ASC
Keyword used in the ORDER BY clause to sort results in ascending (A-Z, 0-9) order. This is the default sort order if not specified.
AVG()
An aggregate function that calculates the average value of a numeric column.
BETWEEN
A logical operator used in the WHERE clause to filter values within a specified range (inclusive).
CASE Statement
A conditional expression in SQL that allows you to define different outcomes based on various conditions, similar to 'if-then-else' logic in programming.
Column
A vertical entity in a table that contains all the data for a specific attribute. Also known as a field.
Common Table Expression (CTE)
A temporary, named result set that you can reference within a SELECT, INSERT, UPDATE, or DELETE statement. Defined with the WITH clause, CTEs improve query readability and modularity.
COUNT()
An aggregate function that returns the number of rows that match a specified criterion. `COUNT(*)` counts all rows, while `COUNT(column_name)` counts non-NULL values in a column.
CRUD
An acronym for Create, Read, Update, Delete. These are the four basic functions of persistent storage in database applications, corresponding to INSERT, SELECT, UPDATE, and DELETE SQL commands.
Database
An organized collection of structured information, or data, typically stored electronically in a computer system. It can be accessed, managed, modified, updated, controlled, and organized.
Data Type
A classification that specifies what kind of value a column can hold (e.g., INTEGER, TEXT, REAL, BOOLEAN, DATE, DATETIME).
DDL (Data Definition Language)
SQL commands used to define, modify, or drop database structures (tables, views, indexes). Examples: CREATE TABLE, ALTER TABLE, DROP TABLE.
DML (Data Manipulation Language)
SQL commands used to manage data within database objects. Examples: INSERT, UPDATE, DELETE, SELECT.
DESC
Keyword used in the ORDER BY clause to sort results in descending (Z-A, 9-0) order.
DISTINCT
Keyword used with SELECT to return only unique (non-duplicate) values in the specified column(s).
Foreign Key (FK)
A column or set of columns in one table that refers to the Primary Key in another table. It establishes a link between the two tables, enforcing referential integrity.
FROM Clause
Specifies the table(s) from which to retrieve data in a SELECT statement.
GROUP BY Clause
Used with aggregate functions to group rows that have the same values in specified columns into a summary row for each group.
HAVING Clause
Used with the GROUP BY clause to filter groups based on aggregate conditions. Similar to WHERE, but applies to groups, not individual rows.
INNER JOIN
Returns only the rows that have matching values in both tables being joined.
INSERT INTO
A DML command used to add new rows of data into a table.
JOIN
A SQL clause used to combine rows from two or more tables, based on a related column between them.
LAG()
A window function that accesses a row at a given physical offset before the current row within a partition. Useful for comparing a row's value to a previous row's value.
LEFT JOIN (LEFT OUTER JOIN)
Returns all rows from the left table, and the matching rows from the right table. If there is no match, NULLs are returned for columns from the right table.
LIMIT Clause
Used to constrain the number of rows returned by a query. Common in MySQL, SQLite, and PostgreSQL.
LIKE Operator
Used in the WHERE clause to search for a specified pattern in a column. Often used with wildcard characters (% for zero or more characters, _ for a single character).
MAX()
An aggregate function that returns the maximum value in a column.
MIN()
An aggregate function that returns the minimum value in a column.
NULL
A marker indicating that a data value does not exist in the database. It is not equivalent to zero, an empty string, or false.
ORDER BY Clause
Used to sort the result-set of a query in ascending (ASC) or descending (DESC) order.
Primary Key (PK)
A column or set of columns in a table that uniquely identifies each row in that table. It must contain unique values and cannot contain NULL values.
Query
A request for data or information from a database. Usually refers to a SELECT statement.
RANK()
A window function that assigns a unique rank to each row within its partition based on the order specified. If rows have identical values, they receive the same rank, and the next rank is skipped (e.g., 1, 1, 3).
Relational Database
A type of database that stores and provides access to data points that are related to one another. Relational databases are based on the relational model, which organizes data into tables (relations).
ROW_NUMBER()
A window function that assigns a unique, sequential integer to each row within its partition, starting at 1 for the first row in each partition. It does not skip numbers for identical values.
Row
A horizontal entity in a table representing a single record or entry. Also known as a record or tuple.
Schema
The logical design of a database, including table names, column names, data types, relationships between tables, and constraints.
SELECT Statement
The most common SQL DML command, used to retrieve data from one or more tables in a database.
SQL (Structured Query Language)
A standard language for storing, manipulating, and retrieving data in relational databases.
SQLite
A C-language library that implements a small, fast, self-contained, high-reliability, full-featured, SQL database engine. It is the most widely deployed database engine in the world.
Subquery
A query (SELECT statement) nested inside another SQL query. It can return a single value, a single row, a single column, or a table.
SUM()
An aggregate function that calculates the sum of all values in a numeric column.
Table
A collection of related data held in a structured format within a database. It consists of columns and rows.
UNION
A set operator that combines the result-set of two or more SELECT statements into a single result-set. Each SELECT statement must have the same number of columns and compatible data types.
UPDATE
A DML command used to modify existing records in a table.
WHERE Clause
Used to filter records based on specified conditions. It appears after the FROM clause and before GROUP BY or ORDER BY.
Window Function
A function that performs a calculation across a set of table rows that are somehow related to the current row. Unlike aggregate functions, window functions do not group rows into a single output row; they retain individual rows.
